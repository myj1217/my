hoisting이 뭐냐?


hoisting 현상 (직역: 끌어올리다)

  • 변수를 어느 위치에서 선언하더라도 사용할 수 있음을 의미한다 - 자바스크립트에서 지원
  • var, let, const로 선언한 변수 모두 hoisting을 지원한다

var 와 let의 차이 2

var name1 = "chul";
var name1 = "soo";
console.log(name1);
let name2 = "chul";
let name2 = "soo";  // name2 = "soo"; 로 변경하면 정상 출력 -> "soo"
console.log(name2);

  • var은 재선언이 가능하므로 통과됨 - 나중에 문제 발생 - 이래서 var 안씀
  • let은 재선언이 불가능하므로 통과되지 않음 (재할당은 가능) - 이래서 let 씀
  • (번외편) const는 재선언, 재할당 불가능 - 상수이기 때문

대부분의 프로그래밍 언어가 ‘블록 레벨 스코프’라는 것을 따른다
javascript의 경우에는
var는 ‘함수 레벨 스코프’라는 것을 따르고,
let, const는 ‘블록 레벨 스코프’라는 것을 따른다

  • == : 값만 같으면 true

reduce 메서드

function sum(a,b, ...args){
  let result = 0;
  if(a != undefined){result = a;}else{return 0;}
  if(b != undefined){result += b;}
  result += args.length > 0 ? args.reduce((subsum, arg)=>subsum+=arg) : 0;
  return result;
}
console.log(sum(1,2,3,4,5));
console.log(sum(1,2));
console.log(sum(1));

reduce() 메서드는 배열의 각 요소에 대해 주어진 리듀서 (reducer) 함수를 실행하고, 하나의 결과값을 반환합니다.

const sum = (a=0, b=0, ...args) => {

};

이렇게 단 세줄로 요약할 수도 있다

const sum = (a=0, b=0, ...args) => {

};
